⚠️ VERIFICATION SYSTEM - MAJOR CHANGE
═══════════════════════════════════════════════════════════════

This is the BIGGEST change in the entire system!

OLD SYSTEM: 5-Digit OTP in Separate Boxes
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 5 separate input boxes
- User types one digit per box
- Auto-advance between boxes
- Paste feature splits digits
- 60-second timer
- File: otp.html (5 <input> elements)
- Example: [1] [2] [3] [4] [5]

NEW SYSTEM: Single Link Input
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 1 single text input box
- User pastes entire verification link
- System extracts code from link
- Multiple link formats supported
- 30-second timer ⚠️ (REDUCED FROM 60s)
- File: otp.html (1 <input type="text"> element)
- Example: [Paste verification link here...]

Supported Link Formats:
1. Full URL: https://maxitbw.co.bw/verify?code=ABC123
2. Path only: /verify/ABC123
3. Code only: ABC123

File Changes:
- otp.html: Removed 5 boxes, added single input
- otp-script.js: Changed validation from 5 digits to link parsing
- style.css: Removed .otp-boxes, added .link-input styles

═══════════════════════════════════════════════════════════════
⏱️ TIMER CHANGE - CRITICAL
═══════════════════════════════════════════════════════════════

OLD: 60 seconds (1 minute)
NEW: 30 seconds ⚠️

This affects:
- Initial countdown: 30s instead of 60s
- Resend timer: 30s instead of 60s
- Circle animation: maxTime = 30
- Warning threshold: < 10s (was < 20s)

Files changed:
- otp-script.js: timeLeft = 30, resendTimeLeft = 30
- otp.html: countdown display starts at 30

Reason for change:
- Max it BW uses shorter verification windows
- Industry standard for link-based verification
- Increases security
